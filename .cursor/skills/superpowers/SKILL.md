# Superpowers

该技能用于让代理在执行工程任务时表现得更稳定、更全面：先快速定位上下文，再并行推进修改、验证和交付。

## 何时使用

- 任务涉及多步骤实现（探索、修改、测试、提交、PR）。
- 需要降低回归风险（变更前后都要验证）。
- 需要在信息不完整时快速自助排障，而不是停在方案讨论。

## 核心能力

1. **并行探索**
   - 使用高信噪比搜索先定位关键文件、入口函数和调用链。
   - 可并行收集上下文，缩短定位时间。

2. **面向交付的实现**
   - 在最小侵入前提下完成需求，而非过度重构。
   - 保持改动原子化，必要时拆成多个小提交。

3. **严格验证**
   - 至少执行与改动相关的验证（测试、构建或静态检查）。
   - 验证失败时先自助修复，再重新验证。

4. **端到端收尾**
   - 清晰记录改动内容与验证结果。
   - 准备可直接评审的提交与 PR 描述。

## 执行清单

1. 明确目标与约束（分支、兼容性、边界）。
2. 搜索并阅读最小必要上下文。
3. 实施改动，保持命名和风格一致。
4. 运行验证并处理失败。
5. 提交代码并推送分支。
6. 更新或创建 PR（如工作流要求）。

## 质量门槛

- 不引入与需求无关的改动。
- 不回退用户已有未请求修改。
- 对外说明必须包含：改了什么、为什么、如何验证。
